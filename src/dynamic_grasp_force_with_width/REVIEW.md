## 결론

**로봇이 물체 무게를 잽니다.** 아이폰 13(174 g)을 노이즈의 10배 SNR로 검출했습니다.
분해능 **20~50 g**. 즉 폭으로 힘을 추측(`F = C·w²`)할 필요가 없습니다.

**집고 → 무게 재고 → 힘 계산 → 다시 조이기.** 이게 진짜 2단계입니다.

---

## 1. 무게 측정 — 됩니다

```
빈 그리퍼 닫기 (대조군)   ΔJ3 = -0.137 Nm
폰 물고 닫기              ΔJ3 = -0.686 Nm
───────────────────────────────────────
폰만의 효과               ΔJ3 = -0.549 Nm      ← 이론 예상 0.5 Nm
노이즈                            0.055 Nm      ← 신호가 노이즈의 10배
```

### 왜 처음엔 안 됐나 — 데이터 경로가 문제

**E0509 에는 관절토크 센서도 F/T 센서도 없습니다.**
`raw_joint_torque` 가 사실 `gravity_torque` 와 **완전히 동일한 모델 계산값**이라,
`external_joint_torque = raw − gravity = 0` 이 **구조적으로 항상 성립**합니다.

| 경로 | |
|---|---|
| `get_tool_force()` | ❌ 위 필드를 읽으므로 **영원히 0** |
| `get_workpiece_weight()` | ❌ 측정 함수가 아님 (`set_` 으로 선언한 값을 돌려주는 게터) |
| `raw_force_torque` | ❌ F/T 센서 미장착 |
| **`actual_motor_torque`** (`read_data_rt`) | ✅ **유일한 실측값. 이걸 쓰세요** |

### 쓰는 법

**툴 등록이 선행돼야 합니다.** 안 하면 중력 모델이 틀려서 아무것도 안 됩니다.
**manual 모드에서만** 됩니다 (autonomous 에서는 거부).

```
1. 상자 폭을 자로 재서 stroke 를 정한다
2. 픽 자세에서 '빈 그리퍼'를 그 stroke 까지 닫고 측정   → 기준선
3. 상자를 물고 '같은 자세·같은 stroke' 에서 다시 측정
4. 질량 = a·Δτ + b        (a, b 는 여러 무게로 회귀해서 구함)
```

> ⚠️ **자세가 바뀌면 지렛대가 바뀝니다.** 기준선과 측정은 반드시 같은 자세에서.
> ⚠️ **기준선은 stroke 마다 다릅니다.** 여닫는 것만으로 0.137 Nm(≈43g) 차이가 납니다.
> 그래서 상자 크기별로 기준선을 맞춰야 합니다 → 아래 **4번 절(B-5)** 참고.

구현: `grasp_v2.ipynb` 의 `WeightSensor`.

## 2. 실행을 막는 버그

| 버그 | 내용 |
|---|---|
| **`is_force_closure()`** | 법선이 `[-dzdx, -dzdy, 1]` 이라 z가 항상 +. antipodal(`|n1+n2| < 0.2`)이 **수학적으로 불가능** → `select_best_grasp()` 항상 `None` → **`run_task()` 100% 실패**. 직육면체 상자엔 과설계이니 **빼세요** |
| **접촉점** | `box[0], box[2]` 는 **대각선 꼭짓점**입니다 (주석은 "마주보는 변의 중점"). 80×40 사각형에서 89.4px(대각선)가 나옵니다. `(box[0]+box[1])/2`, `(box[2]+box[3])/2` 가 맞습니다 |
| **힘의 단위** | `default_force = 20.0` (뉴턴)이 `gripper_close(current=20)` 으로 들어갑니다. 권장 `current` 는 200~400 이라 **20은 상자를 무조건 놓치는 값**입니다. 사실 코드 전체에서 `force` 는 그냥 `current` 로 쓰입니다 — **"N" 이름표만 잘못 붙었습니다** |
| **`stability_score()`** | `contact_points`(mm)와 `center_of_mass`(픽셀)를 뺍니다. 값이 무의미합니다 |

> `grasp_v2` 는 뉴턴을 아예 안 씁니다. `current = A·질량 + B` 로 직접 회귀합니다.
> 물리식은 "필요한 힘이 무게에 **비례**한다"는 **형태**만 주고, 계수는 실험이 줍니다.
> N ↔ current 변환은 알 수 없고, 알 필요도 없습니다.

### ★ B-5 — 기준선이 그리퍼 위치에 의존합니다 (가장 중요한 지적)

**빈 그리퍼를 열었다 닫는 것만으로 토크가 0.137 Nm 바뀝니다** (≈ **43 g** 상당, 아이폰의 25%).
손가락이 움직이면 무게중심이 이동하기 때문입니다.

**상자 크기가 다르면 손가락이 멈추는 위치도 다릅니다.** 즉 **상자마다 기준선이 달라집니다.**
v2 초안은 기준선 하나만 잡았으므로 **상자 크기가 바뀌면 무게가 틀립니다.**

**해법** — 우리 그리퍼는 현재 위치를 **읽을 수 없으므로**
(`GripperCmd.srv` = 제어 전용, 피드백 없음), 이렇게 우회합니다.

```
상자 폭을 자로 잰다  →  그 폭에 해당하는 stroke 로 '명령'한다
                     →  빈 그리퍼도 같은 stroke 로 닫아 기준선을 잡는다
                     →  두 측정의 손가락 위치가 같아진다 ✓
```

즉 **"완전히 닫아라(position 0)"가 아니라 "이 stroke 까지 닫아라"** 로 바꿉니다.

> 그래서 **오늘 절차에 "상자 폭을 자로 재기"와 "기준선 테이블 수집(30분)"이 추가**됐습니다.

### ★ B-1 — 캘리브레이션을 다중점 회귀로

v2 초안의 `calibrate()` 는 **1점으로 원점 지나는 비율**만 구했습니다 (순환논리).
**여러 무게로 `질량 = a·Δτ + b` 를 회귀**하도록 바꿨습니다 (오프셋 `b` 포함).
`R²`, 잔차표, LOO-CV 를 출력해서 **선형성을 확인**합니다.

`R² < 0.95` 면 원인 후보를 알려줍니다 — 처짐에 따른 지렛대 변화 / 그리퍼 개도 차이 / 모터토크 포화.

### 나머지 반영

| 지적 | 조치 |
|---|---|
| A-1 서비스 결과 미검증 | `_call()` 이 `success=False` / timeout 이면 **예외** |
| A-2 노이즈 경고가 경고로만 끝남 | `measure()` 가 `(mass, ok)` 반환 → `ok=False` 면 **fixed 로 자동 폴백** |
| A-3 조용한 clamp | MIN 미만은 정보, **MAX 초과는 위험 경고** + `status="over_max"` 반환 |
| A-4 `PICK_POS` 휘발 | `pick_pose.json` 저장/로드 |
| A-5 모드 전환 미확인 | `set_mode()` 가 **폴링해서 확인**, 안 되면 예외 |
| A-6 타임스탬프 없음 | 모든 기록에 `t` 추가 |
| A-7 held-out 없음 | 데이터가 3~4점이라 held-out 은 무리 → **LOO-CV** 로 대체 |
| C-1 이동 중 미끄러짐 | **들어올린 뒤 재측정**. 질량이 절반 아래로 떨어지면 "놓쳤습니다" |
| B-4 다중관절 | `full_dtau`(6축 전부)를 **미리 기록**만 해둠 → 나중에 데이터로 판단 |

### ⚠️ D-1 (점진적 그립) 의 하드웨어 제약

동료분이 모르실 수 있는 사실입니다.

- **그리퍼 현재 위치를 읽을 수 없습니다.** 슬립 신호로 position 을 쓰려면
  `gripper_service` 에 **Modbus 읽기(FC03, Present Position = 611)** 를 추가해야 합니다.
- 모터 토크는 쓸 수 있지만 **한 번 재는 데 약 2초** 걸립니다 (40샘플 × 0.05초).
  이동 중 실시간 감지엔 느립니다. 샘플을 줄이면 노이즈가 커지고요.
  **이 트레이드오프를 실측으로 정해야** 합니다.

**저울로 잰** 물체 **3~4개**로 다중점 회귀합니다. 물체마다:

```python

```

전부 모았으면:

```python

```

| 증상 | 대처 |
|---|---|
| **R² < 0.95** | 선형이 아닙니다. 셀 출력이 원인 후보를 알려줍니다 |
| 잔차표에 ⚠ 표시 | 그 점의 오차가 15% 넘습니다. 저울값·폭을 다시 확인 |
| "노이즈의 3배 미만" 경고 | 물체가 매달려 있지 않거나 너무 가볍습니다 |
| 값이 계속 튐 | 측정 중 로봇을 건드리지 마세요 |
