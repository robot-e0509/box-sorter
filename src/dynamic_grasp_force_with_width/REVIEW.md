# 코드 리뷰

수정본은 [`grasp_v2.ipynb`](grasp_v2.ipynb) 로 따로 만들었습니다. 원본은 그대로 뒀습니다.

## 결론

**로봇이 물체 무게를 잽니다.** 아이폰 13(174 g)을 노이즈의 10배 SNR로 검출했습니다.
분해능 **20~50 g**. 즉 폭으로 힘을 추측(`F = C·w²`)할 필요가 없습니다.

**집고 → 무게 재고 → 힘 계산 → 다시 조이기.** 이게 진짜 2단계입니다.

---

## 1. 무게 측정 — 됩니다

```
빈 그리퍼 닫기 (대조군)   ΔJ3 = -0.137 Nm
폰 물고 닫기              ΔJ3 = -0.686 Nm
───────────────────────────────────────
폰만의 효과               ΔJ3 = -0.549 Nm      ← 이론 예상 0.5 Nm
노이즈                            0.055 Nm      ← 신호가 노이즈의 10배
```

### 왜 처음엔 안 됐나 — 데이터 경로가 문제

**E0509 에는 관절토크 센서도 F/T 센서도 없습니다.**
`raw_joint_torque` 가 사실 `gravity_torque` 와 **완전히 동일한 모델 계산값**이라,
`external_joint_torque = raw − gravity = 0` 이 **구조적으로 항상 성립**합니다.

| 경로 | |
|---|---|
| `get_tool_force()` | ❌ 위 필드를 읽으므로 **영원히 0** |
| `get_workpiece_weight()` | ❌ 측정 함수가 아님 (`set_` 으로 선언한 값을 돌려주는 게터) |
| `raw_force_torque` | ❌ F/T 센서 미장착 |
| **`actual_motor_torque`** (`read_data_rt`) | ✅ **유일한 실측값. 이걸 쓰세요** |

### 쓰는 법

**툴 등록이 선행돼야 합니다.** 안 하면 중력 모델이 틀려서 아무것도 안 됩니다.
**manual 모드에서만** 됩니다 (autonomous 에서는 거부).

```
1. 픽 자세에서 빈 그리퍼를 '닫고' actual_motor_torque 측정   → 기준선
2. 상자를 물고 '같은 자세'에서 다시 측정
3. 질량 = (측정값 − 기준선) × 계수        (실측 기준 약 317 g/Nm)
```

> ⚠️ **자세가 바뀌면 지렛대가 바뀝니다.** 기준선과 측정은 반드시 같은 자세에서.
> ⚠️ **기준선은 "닫은" 상태로.** 여닫는 것만으로 0.14 Nm 차이가 납니다.

구현: `grasp_v2.ipynb` 의 `WeightSensor`.

## 2. 실행을 막는 버그

| 버그 | 내용 |
|---|---|
| **`is_force_closure()`** | 법선이 `[-dzdx, -dzdy, 1]` 이라 z가 항상 +. antipodal(`|n1+n2| < 0.2`)이 **수학적으로 불가능** → `select_best_grasp()` 항상 `None` → **`run_task()` 100% 실패**. 직육면체 상자엔 과설계이니 **빼세요** |
| **접촉점** | `box[0], box[2]` 는 **대각선 꼭짓점**입니다 (주석은 "마주보는 변의 중점"). 80×40 사각형에서 89.4px(대각선)가 나옵니다. `(box[0]+box[1])/2`, `(box[2]+box[3])/2` 가 맞습니다 |
| **힘의 단위** | `default_force = 20.0` (뉴턴)이 `gripper_close(current=20)` 으로 들어갑니다. 권장 `current` 는 200~400 이라 **20은 상자를 무조건 놓치는 값**입니다. 사실 코드 전체에서 `force` 는 그냥 `current` 로 쓰입니다 — **"N" 이름표만 잘못 붙었습니다** |
| **`stability_score()`** | `contact_points`(mm)와 `center_of_mass`(픽셀)를 뺍니다. 값이 무의미합니다 |

> `grasp_v2` 는 뉴턴을 아예 안 씁니다. `current = A·질량 + B` 로 직접 회귀합니다.
> 물리식은 "필요한 힘이 무게에 **비례**한다"는 **형태**만 주고, 계수는 실험이 줍니다.
> N ↔ current 변환은 알 수 없고, 알 필요도 없습니다.

## 4. 데이터 수집 (본 게임) — 2시간

**상자 하나마다** 이걸 반복합니다.

```python
try_current(150)     # 낮은 값부터. 무게는 자동 기록됨
# → 들어올렸을 때 미끄러지는지 눈으로 확인

reject(0)            # 미끄러졌으면
try_current(200)
reject(1)
try_current(250)
accept(2)            # 버텼으면 → 버틴 것 중 '가장 낮은' 값을 accept
```

**핵심: "버틴 것 중 가장 낮은 current"** 를 accept 하세요.
아무 성공값이나 accept 하면 회귀가 과대평가됩니다.

그다음 **무게를 바꿔서** 반복합니다. 최소 3종, 많을수록 좋습니다.

> 상자가 찌그러지기 시작하는 `current` 도 기록해두세요 → `CURRENT_MAX` 를 그 아래로 낮춥니다.
