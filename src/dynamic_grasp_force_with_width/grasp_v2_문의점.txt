grasp_v2 노트북 — 추가 개선점 정리
(REVIEW.md 이후 논의 내용, 2026-07-14)

=================================================
A. 코드적 문의점
=================================================

1. register_tool() 결과 미검증
   - ok_tool = _call(tool_cli, r) / ok_tcp = _call(tcp_cli, r) 를 받아놓고
     성공 여부를 한 번도 체크 안 함. 서비스 콜 실패해도 그냥 다음 줄로 진행됨.

2. measure() 의 노이즈 경고가 경고로만 끝남
   - "변화가 노이즈의 3배 미만" 경고를 출력만 하고, 신뢰 못 할 값을
     pick_with_weight_feedback() 이 그대로 받아서 힘 계산에 씀.
   - 개선안: 경고 시 fixed 모드로 자동 폴백 검토.

3. clamp_current() 가 조용히 잘라버림
   - CURRENT_MIN~MAX 밖으로 나가도 경고 없이 clamp만 함.
   - 특히 MAX 초과는 "이 물체는 지금 설정으로 못 버틸 수 있다"는
     위험 신호인데 그대로 진행돼버림. MIN 미만과 성격이 다른데 구분 없음.
   - GraspPlanner.current_for() 에도 동일 패턴 반복됨 (원인 하나, 발생 두 곳).
   - 논의된 대안: soft/hard limit 2단계로 나누기, 또는 근본적으로
     한 번에 계산값으로 밀어붙이지 않고 점진적 그립으로 전환 (아래 D 참고).

4. PICK_POS 가 파일로 저장 안 됨
   - 메모리 변수라 커널 재시작하면 날아감. weight_calib.json / 
     grasp_trials_v2.json 처럼 pick_pose.json 으로 저장 필요.

5. 모드 전환(MANUAL/AUTONOMOUS)을 확인 없이 진행
   - set_robot_mode() 후 wait(1) 만 하고 실제 전환 완료 여부를 안 봄.
   - register_tool(), 5번 티칭 셀 둘 다 동일 패턴.

6. trial 기록에 타임스탬프 없음
   - 나중에 드리프트 의심될 때 "언제 잰 데이터인지" 추적 불가.

7. 회귀 검증 데이터셋 분리 없음
   - fit() 에 쓴 데이터로만 RMSE 계산. held-out 데이터로 검증하는
     절차가 없어서 과적합 여부를 모름.


=================================================
B. 실측이 필요한 것
=================================================

1. 캘리브레이션 다중점 회귀 + 선형성 검증  ★ 우선순위 높음
   - 지금 calibrate() 는 known_mass 하나로 grams_per_nm 비율만 구함
     (원점을 지난다고 가정, 오프셋 없다고 가정).
   - 여러 무게(최소 3~4점)로 찍어서 mass = a·Δτ + b 형태로 회귀 필요.
   - 산점도/잔차로 실제 선형인지 확인. 비선형이면 원인 후보:
     처짐에 따른 지렛대 변화, 그리퍼 개도 변화, 모터 토크-전류 포화 등.
   - 힘 회귀(current = A·M + B) 도 동일하게 선형 가정 중이라
     같이 검증 필요.

2. CURRENT_MAX (상자 찌그러지는 값)
   - 현재 TODO. 데이터 채우면서 자연히 채워질 항목.

3. CURRENT_PROBE 적정성
   - 지금 100으로 고정값인데, 물체마다(무게·재질) 다를 수 있음.
     너무 약하면 무게 측정 자체가 불안정, 너무 강하면 "약하게 집기"
     의미가 없어지거나 가벼운 물체는 그 자체로 눌릴 수 있음.

4. WEIGHT_JOINT — 다중 관절 활용 여부
   - 지금은 J3 하나만 사용. 픽 자세가 고정이면 지금은 급하지 않음
     (백로그로 내려도 됨).
   - 다만 여러 관절을 동시에 활용하면 SNR 개선 + 이상치 교차검증
     가능 (센서 퓨전 개념). motor_torque() 는 이미 전체 관절 값을
     반환하니, 다음 실측 때 전체 관절 Δτ/노이즈를 같이 로깅해두면
     나중에 결합 여부를 데이터로 판단 가능.

5. 물체(상자) 크기별 자세 차이 문제  ★ 구조적 이슈
   - 상자 크기가 다르면 그리퍼 닫힘 위치(position)도 달라지고,
     접근 각도도 달라질 수 있어 "같은 자세" 전제가 깨질 수 있음.
   - 다루는 상자 크기 편차가 실제로 큰지부터 확인 필요.
   - 대응 후보: (a) 그리퍼 개도별 기준선 테이블, (b) 오픈 상태 기준
     기준선 + 모델 보정, (c) 상자 크기군별 별도 캘리브레이션.


=================================================
C. 실측 + 코드 보완 고려점
=================================================

1. 이동 중(들어올리는 동안) 미끄러짐 검증 로직 없음
   - try_current() 실험 단계에선 사람이 눈으로 확인하지만,
     실제 운용(pick_with_weight_feedback)에는 확인 단계가 아예 없음.
   - 정지 상태보다 이동 중(가속/감속/방향전환)이 필요한 힘이 더 커질
     수 있어 더 위험한 구간.
   - 개선안: 들어올린 후 scale.measure() 재확인, 혹은 그리퍼 position
     값으로 물체 유무 확인.


=================================================
D. 실측하며 다듬을 것
=================================================

1. 점진적 그립(progressive grip) + 실시간 슬립 감지
   - 지금: 계산된 current 값으로 한 번에 조임 (open-loop).
   - 대안: CURRENT_PROBE 에서 시작해 서서히 올리며 슬립 여부를
     실시간 체크, 안정되면 멈춤 (closed-loop).
   - 이게 되면 clamp 문제(계산값이 부정확해도 실시간 보정됨)와
     이동 중 슬립 문제를 동시에 해결할 가능성.
   - 실측하면서 정해야 할 것:
     - 슬립 신호로 뭘 쓸지 (토크 변화 / position 변화 / 기타)
     - 루프 주기가 실시간 대응 가능한 속도인지 (지금 wait(1) 기준)
     - CURRENT_MAX 까지 올려도 안 멈추면 어떻게 할지 (중단/재시도 기준)
   - try_current() 실험에서 이미 사람이 이 과정을 수동으로 하고
     있으니, 그 데이터가 step 크기 설계에도 재활용 가능.


=================================================
우선순위 요약
=================================================
1순위 (다음 실측 세션): 다중점 회귀 + 선형성 검증, CURRENT_MAX,
       CURRENT_PROBE, 상자 크기별 자세 차이 여부 확인
2순위: 점진적 그립 실험 (신호/주기/실패기준 탐색)
3순위(백로그): 다중 관절 결합, WEIGHT_JOINT 자동화
코드 패치(로봇 불필요, 아무 때나 가능): A 항목 전체
